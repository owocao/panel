# biu-panel V1.0 任务拆解清单 v0.1

## 1. 阶段原则

V1.0 采用先文档、再原型、再后端、再联调、最后 Docker 交付的顺序。

开发阶段不使用 Docker 作为日常调试环境，避免增加调试复杂度。Docker 只在 V1.0 功能稳定后用于交付验证。

每个阶段完成后应更新文档，并将重要文档同步到思源笔记。

## 2. 里程碑

### M0：项目准备

目标：建立项目结构和开发约束。

任务：

- 创建基础目录结构。
- 初始化 Git 仓库。
- 编写 `.gitignore`。
- 固化 Node 版本建议文件，例如 `.nvmrc` 或 `.node-version`。
- 初始化后端 Go 模块。
- 初始化前端 Vue 3 + Vite 项目。
- 确认本机开发启动方式。

验收：

- 可以在本机分别启动后端和前端。
- 项目不会把依赖、数据库、上传文件、备份文件提交到 Git。

### M1：产品原型与页面骨架

目标：先把主要页面交互搭起来，避免后端完成后才发现体验不对。

任务：

- 首页布局原型：分组标题 + Sun-Panel 风格小卡片。
- 右键菜单原型：卡片和分组操作菜单。
- 移动端长按菜单原型。
- 收藏夹左侧抽屉原型。
- 收藏夹抽屉内左目录树 + 右网址列表布局。
- 登录页和首次初始化页原型。
- 设置页原型：站点设置、图片设置、S3 设置、备份恢复。

验收：

- 电脑端和手机端主要页面能正常展示。
- 首页不依赖收藏夹数据加载。
- 收藏夹抽屉打开/关闭体验明确。

### M2：数据库与后端基础

目标：建立稳定的后端基础能力。

任务：

- 设计 SQLite 表结构。
- 实现数据库初始化和迁移机制。
- 实现配置加载：开发环境配置、Docker 环境变量预留。
- 实现统一 API 响应格式。
- 实现基础错误处理和日志。
- 实现静态文件服务预留。

核心数据表建议：

- `users`：管理员账号。
- `login_logs`：登录成功/失败日志。
- `settings`：系统设置。
- `nav_groups`：首页导航分组。
- `nav_items`：首页导航卡片。
- `bookmark_folders`：收藏夹文件夹。
- `bookmarks`：收藏夹网址。
- `assets`：图片、图标、背景等资源。
- `storage_configs`：S3 配置。
- `backup_records`：备份记录。

验收：

- 后端可以自动创建 SQLite 数据库。
- 表结构可重复初始化，不破坏已有数据。
- 基础健康检查接口可访问。

### M3：登录与初始化

目标：完成第一版安全入口。

任务：

- 首次打开初始化管理员账号。
- 支持通过部署环境变量初始化账号。
- 实现登录、登出、当前用户状态接口。
- 实现浏览器关闭后失效的默认会话。
- 实现“记住登录”。
- 实现登录成功/失败日志。
- 实现连续失败 5 次锁定 15 分钟。

验收：

- 未初始化时只能进入初始化页。
- 初始化后可以登录进入首页。
- 登录失败锁定规则生效。

### M4：首页导航功能

目标：完成首页导航核心功能。

任务：

- 导航分组增删改查。
- 导航分组折叠/展开。
- 导航分组拖拽排序并自动保存。
- 导航卡片增删改查。
- 导航卡片拖拽排序并自动保存。
- 卡片右键菜单。
- 移动端长按菜单。
- 本地图片、图片链接作为图标。
- 内网/外网地址切换。
- 自动判断内外网：统一检测地址、超时时间、单卡片覆盖。

验收：

- 首页首屏加载快，不请求收藏夹树。
- 卡片默认当前标签页打开。
- 分组和卡片排序刷新后保持。
- 内外网切换规则符合需求文档。

### M5：图片与 S3 存储

目标：完成资源管理能力。

任务：

- 本地上传图片。
- 图片链接保存。
- S3 配置页面。
- S3 上传实现。
- 图片来源记录：本地、外链、S3。
- 普通图标/Logo 限制常见图片格式和 5MB。
- 背景图片限制图片类型但不限制大小。

验收：

- 图标、Logo、背景图可以使用本地、外链、S3 资源。
- S3 配置保存后能上传测试图片。

### M6：收藏夹基础功能

目标：完成收藏夹核心管理。

任务：

- 收藏夹抽屉懒加载。
- 文件夹按需加载。
- 文件夹增删改查。
- 网址收藏增删改查。
- 网址纯文本备注。
- 网址 favicon 默认显示且可配置。
- 文件夹和网址拖拽排序。
- 跨文件夹拖拽移动。
- 批量删除。
- 批量移动。
- 收藏夹操作菜单。

验收：

- 打开首页不会加载收藏夹完整数据。
- 点开文件夹才加载子级内容。
- 拖拽后立即保存。
- 删除前有二次确认。

### M7：标题和 favicon 抓取

目标：提升新增收藏的体验。

任务：

- 前端优先抓取标题和 favicon。
- 前端失败后调用后端兜底抓取。
- 后端抓取支持超时。
- 抓取失败允许手动填写。

验收：

- 输入 URL 后能自动填充标题和 favicon。
- 抓取失败不影响保存。

### M8：收藏夹搜索

目标：实现收藏夹模糊搜索。

任务：

- 后端收藏夹搜索接口。
- 搜索范围只包含网址收藏。
- 搜索结果显示标题、URL、favicon、所在路径。
- 不显示文件夹结果。

验收：

- 不打开完整收藏夹树也可以搜索。
- 搜索结果可点击打开。

### M9：收藏夹设为首页卡片

目标：打通收藏夹与首页导航。

任务：

- 收藏夹条目复制为首页卡片。
- 设为首页时选择已有分组。
- 设为首页时允许新建分组。
- 首页卡片与原收藏后续互不影响。

验收：

- 删除首页卡片不影响收藏夹原条目。
- 修改收藏夹条目不影响已复制的首页卡片。

### M10：浏览器书签导入导出

目标：兼容 Chrome 和 Safari 书签文件。

任务：

- 实现 Netscape Bookmark HTML 解析。
- 适配 Chrome 导入。
- 适配 Safari 导入。
- 保留浏览器根目录为顶层文件夹。
- 单次导入内部不去重。
- 后续导入与当前系统已有收藏比较，重复项默认跳过。
- 导入结果展示成功、跳过、失败统计。
- 实现 Chrome/Safari 兼容导出。

验收：

- Chrome 导出的书签可导入并保持结构。
- Safari 导出的书签可导入并保持结构。
- 导出的 HTML 能被 Chrome/Safari 识别。
- 首页导航卡片不进入浏览器书签导出。

### M11：系统设置与个性化

目标：完成个人化配置。

任务：

- 站点标题设置。
- Logo 设置。
- 背景图设置。
- 背景色设置。
- 背景图 cover 显示。
- 收藏夹 favicon 显示开关。
- 内外网自动判断配置。

验收：

- 设置保存后刷新仍生效。
- 设置变更不影响已有导航和收藏夹数据。

### M12：备份与恢复

目标：完成基本可维护性能力。

任务：

- 手动生成 `.tar.gz` 备份包。
- 备份 SQLite 数据库。
- 备份本地上传文件。
- 写入配置和版本信息。
- 下载备份包。
- 上传备份到 S3。
- 恢复备份。
- 版本不一致时警告但允许继续。

验收：

- 下载备份后可以在新环境恢复。
- 恢复前二次确认。
- 恢复会覆盖当前数据。

### M13：测试与稳定性

目标：保证关键路径可靠。

任务：

- 后端单元测试。
- 书签 HTML 导入导出测试。
- 内外网地址选择逻辑测试。
- 备份恢复逻辑测试。
- 前端关键组件测试或手动验收清单。
- 移动端适配检查。

验收：

- 核心测试通过。
- Chrome/Safari 导入导出有样例测试文件。
- 移动端主要操作可用。

### M14：Docker 交付

目标：最终可部署。

任务：

- 编写 Dockerfile。
- 编写 `docker-compose.yml`。
- 编写 `docker run` 示例。
- 编写反向代理说明。
- 验证默认端口 `55088`。
- 验证数据目录分开挂载。
- 编写升级和备份说明。

验收：

- 使用 docker-compose 可启动完整服务。
- 数据落在挂载目录。
- 重建容器不丢数据。

## 3. 推荐开发顺序

建议顺序：

1. M0 项目准备。
2. M1 产品原型与页面骨架。
3. M2 数据库与后端基础。
4. M3 登录与初始化。
5. M4 首页导航功能。
6. M6 收藏夹基础功能。
7. M7 标题和 favicon 抓取。
8. M8 收藏夹搜索。
9. M9 收藏夹设为首页卡片。
10. M5 图片与 S3 存储。
11. M10 浏览器书签导入导出。
12. M11 系统设置与个性化。
13. M12 备份与恢复。
14. M13 测试与稳定性。
15. M14 Docker 交付。

## 4. 第一轮开发建议

第一轮不要急着实现全部功能，建议先完成：

- 项目初始化。
- 首页静态原型。
- 收藏夹抽屉静态原型。
- 登录/初始化静态原型。
- 数据库/API 初稿。

这样可以先确认界面体验和数据结构，再进入完整功能开发。
