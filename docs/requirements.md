# biu-panel V1.0 需求文档 v0.1

## 1. 项目定位

biu-panel 是一个个人使用的 BS 端导航站 + 网页收藏夹系统，参考 Sun-Panel 的导航体验，但不做完全复刻，目标是轻量、快速、简单、适合 Docker 部署和长期自用维护。

核心原则：

- 个人单用户优先，第一版只支持一个管理员账号。
- 后续预留多账号隔离能力，但第一版不实现。
- 首页保持轻量，只加载导航卡片和外部搜索引擎搜索配置，不加载完整收藏夹。
- 收藏夹作为左侧抽屉模块，点击后按需加载。
- 技术路线成熟、轻量、响应快、占用低。

## 2. 技术路线

第一版建议技术路线：

- 后端：Go。
- 前端：Vue 3 + Vite。
- 数据库：SQLite。
- 前端包管理：npm。
- 开发阶段：本机直接运行后端和前端，不使用 Docker 调试。
- 交付阶段：1.0 稳定后补 Dockerfile、docker-compose.yml 和部署文档。

环境约定：

- 项目目录：`/project/panel`。
- 项目名称：`biu-panel`。
- 默认 HTTP 端口：`55088`。
- HTTPS 由反向代理处理，应用本身只提供 HTTP。

## 3. 部署要求

最终交付支持：

- `docker run`。
- `docker-compose.yml`，优先推荐。
- 支持 Nginx Proxy Manager / Nginx / Caddy 等反向代理。
- 优先支持独立域名或子域名方式。
- 不优先支持子路径部署，若后续需要再单独处理。

数据目录采用分开挂载：

- `./data/db:/app/data/db`
- `./data/uploads:/app/data/uploads`
- `./data/backups:/app/data/backups`

## 4. 登录与安全

第一版只支持一个管理员账号。

账号初始化：

- 支持 Docker 环境变量设置初始管理员账号密码。
- 如果没有环境变量，第一次打开网页进入初始化页面。

登录会话：

- 默认浏览器关闭后失效。
- 支持“记住登录”，勾选后长期登录。

登录失败防护：

- 记录登录成功和登录失败日志。
- 连续失败 5 次，锁定登录 15 分钟。
- 第一版不做验证码。

## 5. 首页导航

首页参考 Sun-Panel 的体验，但做更简洁的个人版。

布局：

- 类似 Sun-Panel：分组标题 + 网格小卡片。
- 只做 Sun-Panel 风格的小卡片样式。
- 支持移动端适配。

导航分组：

- 支持新增、编辑、删除。
- 支持拖拽排序。
- 拖拽后立即自动保存。

导航卡片：

- 支持名称、图标、分组、内网地址、外网地址、排序。
- 不需要备注。
- 整张卡片可点击，默认当前标签页打开。
- 如果用户想新标签打开，可使用浏览器右键的新标签页打开。
- 编辑操作放到右键菜单，参考 Sun-Panel。
- 移动端长按打开操作菜单。

首页不做：

- 首页收藏夹内容搜索。首页保留外部搜索引擎搜索入口，用于跳转 Baidu / Google / Bing 等搜索引擎；该搜索不搜索本地收藏夹内容。
- Docker 容器管理。
- 服务器状态监控。
- 网站在线状态检测。

## 6. 内网/外网地址切换

支持全局手动优先级策略。

能力：

- 全局手动切换：内网优先 / 外网优先。
- 支持配置优先地址探测超时时间。
- 内网优先时，优先探测并打开内网地址，超时后回退公网地址。
- 公网优先时，优先探测并打开公网地址，超时后回退内网地址。

建议逻辑：

- 不提供 `lanDetectUrl` / `autoDetectLan` 自动检测配置入口。
- 仅保留 `lanDetectTimeout` 作为优先地址探测超时回退配置。
- 不做统一内网检测地址。
- 不做单卡片独立检测地址，除非后续重新确认进入范围。

## 7. 图片与图标

图标来源：

- 手动上传。
- 图片链接。
- 本地上传后可一键上传至图床。

存储能力：

- 第一阶段支持本地上传。
- 支持 S3 兼容存储。
- 图片记录保存来源：本地、外链、S3。

S3 配置：

- 在页面设置里填写，保存在本地数据库。
- 支持 endpoint、region、bucket、access key、secret key。
- 支持 path-style 开关，兼容 MinIO / R2 等。
- 支持自定义上传目录前缀。
- S3 secret key 不额外加密保存，依赖本地部署和数据库文件权限保护。

上传限制：

- 普通图标：只允许常见图片格式，默认最大 5MB。
- 背景图片：不限制大小，但仍建议限制为图片类型。

## 8. 个性化设置

首页支持：

- 站点标题。
- 背景图。
- 背景色。

背景图显示：

- 第一版只使用常见的 cover 覆盖填充。

背景图片来源：

- 复用图片能力，支持本地上传、图片链接、S3。

## 9. 收藏夹

收藏夹是 biu-panel 区别于普通导航站的核心模块。

入口与加载：

- 首页左侧按钮打开收藏夹抽屉。
- 抽屉从左侧滑出，用完可关闭。
- 首页默认不加载收藏夹内容。
- 点击打开收藏夹后才加载。

布局：

- 左边目录树。
- 右边显示当前文件夹下的网址列表。

目录结构：

- 支持无限层级文件夹。
- 文件夹只有名称和排序，不需要备注。
- 目录树按需加载，点开某个文件夹时才加载子目录。

网址收藏：

- 支持网址、标题、favicon、纯文本备注、排序。
- 输入 URL 后自动抓取网页标题和 favicon。
- 前端优先抓取，失败后后端兜底。
- 抓取失败不影响保存，允许手动填写。
- 默认点击当前标签页打开。

收藏夹列表：

- favicon 可配置显示，默认显示。
- 不记录访问次数。
- 不记录最近访问时间。

拖拽排序：

- 文件夹和网址都支持拖拽排序。
- 支持同级排序。
- 支持跨文件夹拖拽。
- 拖完立即自动保存。

批量操作：

- 支持批量删除。
- 支持批量移动。

收藏夹操作菜单：

- 打开。
- 新标签页打开。
- 新窗口打开。
- 编辑。
- 删除。
- 复制链接。
- 移动到文件夹。
- 设为首页卡片 / 取消首页卡片。
- 批量选择入口。

删除规则：

- 删除前二次确认。
- 确认后永久删除。
- 第一版不做回收站。

## 10. 首页搜索与收藏夹搜索

首页搜索：

- 首页保留外部搜索引擎搜索入口。
- 首页搜索用于跳转到 Baidu / Google / Bing 等搜索引擎。
- 首页搜索不搜索本地收藏夹，不搜索首页导航卡片。
- 搜索引擎列表可在设置中配置。

收藏夹搜索：

搜索范围：

- 模糊搜索仅用于收藏夹。
- 首页导航卡片不需要本地搜索。

搜索方式：

- 收藏夹未打开时不加载目录树。
- 搜索时调用后端搜索接口。
- 搜索结果只显示匹配的网址收藏，不显示文件夹。
- 建议显示所在完整路径，方便区分同名收藏。

## 11. 收藏夹设为首页卡片

规则：

- 收藏夹条目可以设置为首页卡片。
- 设为首页时复制成独立导航卡片。
- 之后收藏夹和首页卡片互不影响。
- 设为首页时可选择已有分组，也可新建分组。
- 首页删除只删除首页导航卡片，不影响收藏夹原收藏。

## 12. 浏览器书签导入导出

目标：

- 兼容 Safari 和 Chrome 的书签导出文件。
- 优先实现通用 Netscape Bookmark HTML 解析。
- 界面可保留 Chrome 导入/导出、Safari 导入/导出选项，便于针对差异适配。

导入规则：

- 保留浏览器根目录，作为顶层文件夹，例如“书签栏”、“其他书签”。
- 单次导入内部不去重，原文件是什么结构、顺序、重复情况，导入后尽量保持一致。
- 后续导入时，只和当前系统已有收藏比较。
- 只有同一个文件夹下 URL、标题、备注/描述等关键字段都一样时，才认为是重复。
- 重复项默认跳过。
- 不同文件夹下即使 URL、标题完全一样，也允许存在。
- 尽量保持原始 URL，不做强标准化，只做必要安全和格式校验。

导出规则：

- 浏览器书签导出只导出收藏夹。
- 首页导航卡片不进入浏览器书签导出。
- 首页导航卡片、系统配置、图片等通过系统备份处理。

## 13. 系统备份与恢复

备份方式：

- 支持手动点击备份，下载备份包。
- V1 不提供备份包上传到 S3 的独立入口。

备份包格式：

- `.tar.gz`。

备份内容：

- SQLite 数据库。
- 本地上传文件。
- 配置和版本信息。
- S3 上的图片默认不下载进备份包，但可以做成可选项。

恢复规则：

- 恢复会覆盖当前数据。
- 二次确认后直接覆盖。
- 备份包记录系统版本。
- 版本不兼容时提示警告，但允许强制恢复。
- 版本不一致时只弹二次确认，不需要输入确认文字。

## 14. 第一版明确不做

第一版不做以下功能：

- 多账号隔离。
- Docker 容器管理。
- 服务器状态监控。
- 首页本地内容搜索。首页外部搜索引擎搜索保留。
- 首页网站在线状态检测。
- 回收站。
- 验证码。
- 复杂权限系统。
- 子路径部署。
- Markdown 备注。
- 标签系统。
